### 2. Submittals API (Shop Drawings & Product Data)
**Purpose**: AI-powered shop drawing review and compliance checking

**Critical Context - Shop Drawings**:
Shop drawings are the MOST CRITICAL submittal type that must be approved before work proceeds. They show exact fabrication/installation details and feed directly into as-built documentation.

**Endpoints Used**:
- `GET /rest/v1.0/submittals` - List submittals
- `GET /rest/v1.0/submittals/{id}` - Get submittal details
- `POST /rest/v1.0/submittals/{id}/responses` - Submit review response
- `PATCH /rest/v1.0/submittals/{id}` - Update submittal status
- `GET /rest/v1.0/submittals/{id}/attachments` - Download shop drawings

**Webhook Events**:
- `submittals.create` - New shop drawing uploaded
- `submittals.update` - Shop drawing status changed

**AI Workflow - Shop Drawing Intelligence**:
1. **Webhook Trigger**: New shop drawing submittal created in Procore
2. **Document Intelligence Agent**: Downloads shop drawing PDF
3. **Drawing Analysis Agent**: 
   - Extracts all objects/components shown (e.g., "50x Type A Manholes, 120x 12" Storm Pipes")
   - Identifies dimensions, materials, connection types
   - Compares against design drawings to detect deviations
4. **Compliance Verification Agent**: 
   - Cross-checks against project specifications
   - Verifies material certifications
   - Checks for code compliance
   - Flags conflicts with other shop drawings (clash detection)
5. **Object Linking**: AI identifies which drawing objects this shop drawing applies to
6. **Quality Scoring Agent**: Assigns confidence score (0-100%)
7. **Result Push**: Posts review findings to Procore submittal
8. **Visual Status Update**: Drawing objects linked to this shop drawing change color (pending â†’ in review)
9. **Notification**: Alerts reviewer in Procore of AI findings with specific issues highlighted

**Shop Drawing Approval Tracking**:
- AI maintains database of which objects have approved shop drawings
- Prevents field work on objects without approved shop drawings
- Alerts if installation photo taken before shop drawing approval
- Tracks shop drawing â†’ field installation â†’ as-built lineage

**Procore Integration Details**:
- Read shop drawing attachments via Documents API
- Post AI review comments to submittal response log
- Update custom fields (e.g., "AI_Shop_Drawing_Score": 92, "Objects_Covered": "MH-1 through MH-15")
- Link shop drawing to specific drawing objects via our object recognition system
- Respect Procore's approval workflow (AI assists, humans approve)

### 4. Drawings API & Markup Integration
**Purpose**: Automated redline creation and as-built generation

**Endpoints Used**:
- `GET /rest/v1.0/drawings` - List drawings
- `GET /rest/v1.0/drawings/{id}` - Get drawing metadata
- `GET /rest/v1.0/drawings/{id}/file` - Download drawing PDF
- `POST /rest/v1.0/drawings` - Upload updated drawing (as-built)
- `POST /rest/v1.0/drawings/{id}/revisions` - Create new revision
- **`POST /rest/v1.0/drawings/{id}/markups`** - Create redline markups
- **`GET /rest/v1.0/drawings/{id}/markups`** - Read existing markups
- **`PATCH /rest/v1.0/drawings/{id}/markups/{markup_id}`** - Update markup

**Webhook Events**:
- `drawings.create` - New drawing uploaded
- `drawings.update` - Drawing revised
- `drawings.markups.create` - New redline added (if available)

**AI Workflow - Automated Redlining**:
1. **Initial Sync**: Download all project drawings from Procore
2. **Drawing Analysis Agent**: Extract symbols, dimensions, coordinate system
3. **Ongoing Redline Generation**:
   - Field photo taken â†’ AI detects change â†’ **auto-creates markup in Procore**
   - Submittal approved â†’ AI extracts modification â†’ **auto-creates markup**
   - Change order issued â†’ AI identifies impact â†’ **auto-creates markup**
4. **Markup Types Generated by AI**:
   - Cloud callouts for changes
   - Dimension annotations
   - Text notes explaining modifications
   - Arrow pointers to affected areas
   - Links to source RFI/submittal/change order
5. **As-Built Compilation**: At closeout, aggregate all Procore redline markups + AI-detected changes
6. **CAD Generation**: Create updated DXF/PDF with all markups compiled
7. **Upload to Procore**: Post final as-built as new drawing revision

**Critical Integration**: 
- AI creates redline markups in Procore's native markup system (not separate layer)
- Uses Procore's "Published Layer" so all team members see AI-generated redlines
- Maintains Procore as single source of truth for all redlines
- Final as-builts appear in Procore's standard Drawings tool with full version history

### Procore SDK Setup

**Backend (Node.js)**:
```javascript
import axios from 'axios';

class ProcoreClient {
  constructor(accessToken, companyId) {
    this.accessToken = accessToken;
    this.companyId = companyId;
    this.baseURL = 'https://api.procore.com/rest/v1.0';
  }

  async makeRequest(method, endpoint, data = null) {
    const config = {
      method,
      url: `${this.baseURL}${endpoint}`,
      headers: {
        'Authorization': `Bearer ${this.accessToken}`,
        'Procore-Company-Id': this.companyId,
        'Content-Type': 'application/json'
      }
    };
    
    if (data) config.data = data;
    
    try {
      const response = await axios(config);
      return response.data;
    } catch (error) {
      // Intelligent retry logic
      if (error.response?.status === 429) {
        // Rate limited, wait and retry
        await this.handleRateLimit(error.response.headers);
        return this.makeRequest(method, endpoint, data);
      }
      throw error;
    }
  }

  // Submittal operations
  async getSubmittal(submittalId) {
    return this.makeRequest('GET', `/submittals/${submittalId}`);
  }

  async postSubmittalResponse(submittalId, response) {
    return this.makeRequest('POST', 
      `/submittals/${submittalId}/responses`, 
      response
    );
  }

  // RFI operations
  async getRFI(rfiId) {
    return this.makeRequest('GET', `/rfis/${rfiId}`);
  }

  async postRFIResponse(rfiId, response) {
    return this.makeRequest('POST', `/rfis/${rfiId}/responses`, response);
  }

  // Drawing operations
  async listDrawings(projectId) {
    return this.makeRequest('GET', `/projects/${projectId}/drawings`);
  }

  async uploadDrawing(projectId, drawingData) {
    return this.makeRequest('POST', 
      `/projects/${projectId}/drawings`, 
      drawingData
    );
  }
}
```

### Rate Limiting Strategy

**Procore API Limits**:
- 3,600 requests per hour per company
- Burst allowance: 10 requests per second

**Our Strategy**:
```javascript
class RateLimiter {
  constructor(maxPerHour = 3000) { // Leave buffer
    this.maxPerHour = maxPerHour;
    this.requestLog = [];
    this.queue = [];
  }

  async throttle() {
    const now = Date.now();
    const oneHourAgo = now - 3600000;
    
    // Clean old requests
    this.requestLog = this.requestLog.filter(t => t > oneHourAgo);
    
    if (this.requestLog.length >= this.maxPerHour) {
      // Wait until oldest request expires
      const waitTime = this.requestLog[0] - oneHourAgo;
      await new Promise(resolve => setTimeout(resolve, waitTime));
      return this.throttle();
    }
    
    this.requestLog.push(now);
  }

  async executeRequest(requestFn) {
    await this.throttle();
    return requestFn();
  }
}
```

### Caching Strategy

**Redis Caching for Procore Data**:
```javascript
class ProcoreCachedClient extends ProcoreClient {
  constructor(accessToken, companyId, redisClient) {
    super(accessToken, companyId);
    this.redis = redisClient;
  }

  async getProjectWithCache(projectId) {
    const cacheKey = `procore:project:${projectId}`;
    
    // Check cache first
    const cached = await this.redis.get(cacheKey);
    if (cached) {
      return JSON.parse(cached);
    }
    
    // Fetch from Procore
    const project = await this.makeRequest('GET', `/projects/${projectId}`);
    
    // Cache for 1 hour (projects don't change often)
    await this.redis.setex(cacheKey, 3600, JSON.stringify(project));
    
    return project;
  }

  async invalidateCache(projectId) {
    await this.redis.del(`procore:project:${projectId}`);
  }
}
```

- **Mitigation**: 
  - Subscribe to Procore developer newsletter
  - Automated integration tests run daily
  - Fallback to graceful degradation if endpoints unavailable
  - Version pinning with controlled upgrades

**Risk: AI accuracy issues damage reputation**
- **Mitigation**:
  - Human-in-the-loop for critical decisions
  - Confidence scores shown with all AI predictions
  - Easy override mechanism
  - Continuous model retraining based on user feedback

**Risk: Procore rate limits impact performance**
- **Mitigation**:
  - Aggressive caching strategy
  - Batch API requests where possible
  - Request rate limiting with queue management
  - Enterprise tier can request higher limits from Procore

### Business Risks

**Risk: Procore builds competing AI features**
- **Mitigation**:
  - Partner with Procore (App Marketplace)
